# Card Flow

## Main flow
raw_source -> episode -> insight / counterview -> research_topic -> hypothesis -> validation -> rule

## Opportunity line
insight -> research_topic -> candidate stocks -> strategy notes

## Learning line
insight -> counterview -> hypothesis -> validation -> rule

## Lineage requirements
Every card should keep references to upstream ids.

## State examples
- research_topic: watchlist, researching, promoted, archived
- hypothesis: pending, validating, confirmed, rejected
- rule: active, weak, invalidated
